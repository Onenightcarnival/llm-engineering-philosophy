# 附录

从主线裁出的深度文章。这些文章的内容本身具有价值，但或因数学门槛较高、或因与相邻章节有内容重叠、或因偏向专题探讨而非主线论证，被从正文中移出。供有兴趣的读者按需深入。

## 来自第二章（不确定性与决策）

- [恐慌时刻的逆向工程](恐慌时刻的逆向工程.md) -- 当新模型发布或框架被废弃时，如何回到第一性原理判断变化是否改变了基本假设。技术领域的"极端恐慌"与金融市场恐慌的结构性同构。
- [知行合一：从设计到实现的一致性](知行合一.md) -- 复杂性偏好、沉没成本谬误、从众心理——知道正确答案与做到正确答案之间的鸿沟，以及将纪律性嵌入工程流程的机制。

## 来自第四章（声明式提示与类型契约，原第三章）

- [提示的可维护性问题](提示的可维护性.md) -- Prompt 作为生产制品面临的可维护性挑战：版本管理、回归测试、重构风险、技术债务。声明式方法如何将这些挑战转化为已有解决方案的问题。
- [Declarative Chain-of-Thought](声明式思维链.md) -- Code as Prompt 的技术实现详解（英文）。结构定义即推理路径、类型系统即约束传播、嵌套结构即递归推理。

## 来自第五章（架构与编排，原第五、六章）

- [状态管理与会话设计](状态管理与会话设计.md) -- 上下文窗口不是数据库。会话状态的分层管理、对话历史的压缩策略、无状态与有状态架构的选择标准。
- [多轮交互的状态机模型](多轮交互的状态机模型.md) -- 用有限状态机的形式化视角审视多轮对话系统设计：显式状态定义、转移函数实现、状态机与 Agent 的控制权差异。

## 来自第六章（测试评估与可观测性，原第八章）
