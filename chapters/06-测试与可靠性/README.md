# 第六章 测试与可靠性

## 核心问题

如何测试一个输出不确定的系统？传统软件测试建立在"相同输入产生相同输出"的假设之上。当这个假设不再成立，测试的含义需要被重新定义。

## 计划文章

- **结构测试与语义测试** -- 结构测试验证"输出是否符合 Schema"，语义测试验证"输出的含义是否正确"。前者可以自动化，后者需要更巧妙的策略。两个层面的测试缺一不可。
- **基于属性的测试在 LLM 应用中的适配** -- Property-based testing 的思想天然适合不确定性系统：不断言具体值，而是断言输出必须满足的性质。如何将 Hypothesis 等工具引入 LLM 应用测试。
- **评估即测试** -- Evaluation 和 Testing 在 LLM 应用中的边界正在模糊。统计度量（准确率、一致性、幻觉率）本质上是一种持续运行的测试套件。
- **可观测性设计** -- 日志、追踪、指标——三大可观测性支柱在 LLM 应用中的具体化。特别是：如何记录 prompt、响应和中间推理步骤，使系统行为可回溯。
- **优雅降级策略** -- 当 LLM 调用失败或质量不达标时，系统应该如何降级？从简单的缓存回退到基于规则的 fallback，不同降级层级的设计。
