# 第六章 测试、评估与可观测性

## 核心问题

如何测试一个输出不确定的系统？传统软件测试建立在"相同输入产生相同输出"的假设之上。当这个假设不再成立，测试的含义需要被重新定义。

前面五章讨论了如何设计和构建 LLM 应用——认识论、决策框架、开发过程、声明式约束、架构与编排。本章回答一个不可回避的问题：怎么知道系统在正常工作？

## 文章

- [不确定性系统的测试哲学](01-不确定性系统的测试哲学.md) -- 结构正确性与语义正确性是两个独立的层面。结构测试已被第四章的类型系统在解析阶段覆盖；测试的真正战场是语义验证和属性测试——特别是那些类型系统无法表达的输入-输出关系、扰动不变性和跨调用一致性。
- [实验管理与统计度量](02-实验管理与统计度量.md) -- Prompt 具有代码与配置的双重身份。它需要版本控制，更需要实验管理：控制变量、效果归因、统计显著性检验。如何定义可量化的质量维度，如何建立具有统计属性的评估基线。
- [从评估到可观测性](03-从评估到可观测性.md) -- 测试与评估的边界在 LLM 应用中消失。评估指标就是测试断言，持续评估就是在线测试套件。从离线评估到在线监控形成一条连续体，可观测性是这条连续体在运行时的延伸——发现"成功的失败"，关联 prompt 版本与输出质量，监控成本。
- [降级设计与数据飞轮](04-降级设计与数据飞轮.md) -- 降级是正常运行的一部分。五级降级层级设计、触发条件的量化、透明度原则。以及如何通过用户反馈、输出日志和人工标注构建正反馈闭环，将降级数据转化为系统改进的燃料。

## 本章的推演逻辑

四篇文章构成一条从测试哲学到运行时保障再到持续改进的推演链：

第一篇建立测试哲学——结构验证是第四章类型系统的直接回报（已解决问题），语义验证和属性测试才是 LLM 测试的真正战场。第二篇引入实验管理——将 prompt 迭代从"凭感觉改"升级为"可量化的实验"，建立版本控制、统计基线和质量度量的纪律。第三篇从离线评估延伸到在线可观测性——测试是上线前的质量门禁，可观测性是上线后的质量监控，两者是同一条线的两端。第四篇讨论最后防线和正反馈——降级设计保证系统在质量不达标时仍能提供服务，数据飞轮将运行数据转化为持续改进的动力。

本章的测试与可靠性方法论回答了"怎么知道系统在正常工作"。[第七章](../07-反模式与陷阱/00-概述.md)则从反面回答"什么做法会让系统出问题"。
