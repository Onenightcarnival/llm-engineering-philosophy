# 第七章 测试与可靠性

## 核心问题

如何测试一个输出不确定的系统？传统软件测试建立在"相同输入产生相同输出"的假设之上。当这个假设不再成立，测试的含义需要被重新定义。

前面六章讨论了如何设计和构建 LLM 应用——认识论、决策框架、prompt 设计、类型约束、架构模式、工作流编排。本章回答一个不可回避的问题：怎么知道系统在正常工作？

## 文章

- [结构测试与语义测试](01-结构测试与语义测试.md) -- 两个独立层面的正确性验证。结构测试（输出是否符合 Schema）可以完全自动化；语义测试（输出的含义是否正确）需要基于规则的断言、参考答案对比和 LLM-as-Judge 等策略。测试的统计性质和测试金字塔的适配。
- [基于属性的测试在 LLM 应用中的适配](02-基于属性的测试.md) -- Property-based testing 天然适合不确定性系统：不断言具体值，而是断言输出必须满足的性质。四类可测试属性（结构、关系、不变性、一致性），以及 API 调用成本下的实践约束。
- [评估即测试](03-评估即测试.md) -- 测试与评估的边界在 LLM 应用中消失了。统计度量（准确率、一致性、幻觉率）本质上是持续运行的测试套件。评估数据集的构建、持续评估的实践、以及评估的成本收益分析。
- [可观测性设计](04-可观测性设计.md) -- 日志、追踪、指标三大支柱在 LLM 应用中的特殊实现。输出质量作为可观测性的新维度，prompt 版本与质量的关联分析，以及成本可观测性。
- [优雅降级策略](05-优雅降级策略.md) -- 降级是正常运行的一部分。五级降级层级设计、触发条件的量化、透明度原则，以及降级与重试的选择标准。

## 本章的推演逻辑

五篇文章构成一条从测试方法到运行时保障的推演链：

第一篇建立测试的分类框架——结构测试和语义测试的独立性、各自的策略和局限性。第二篇引入一种特别适合 LLM 应用的测试范式——基于属性的测试，作为传统基于示例测试的互补。第三篇模糊了测试与评估的边界，将统计度量纳入持续质量保障体系。第四篇从测试转向运行时的可观测性——如果测试是上线前的质量保障，可观测性是上线后的质量监控。第五篇讨论当质量保障失败时的最后一道防线——优雅降级。

本章的测试与可靠性方法论在实践中需要数据科学思维的支撑——统计度量、实验设计、收敛性判据。这正是[第八章](../08-数据科学与计算思维/00-概述.md)的主题。
