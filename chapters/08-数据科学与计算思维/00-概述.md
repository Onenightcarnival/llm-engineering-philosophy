# 第八章 数据科学与计算思维

## 核心问题

数据科学和计算数学训练带来的不仅是具体的方法论，更是一种思维方式：对不确定性的量化直觉、对收敛性的敏感、对实验方法的尊重、对数值稳定性的警觉。这些思维方式在 LLM 应用开发中的价值被严重低估了。

本章是作者学术背景（数据科学硕士、计算数学本科）与工程背景的交汇点。前七章在多处借用了数据科学和计算数学的概念作为类比；本章将这些思维工具系统性地展开，从隐喻提升为方法论。

## 文章

- [提示的版本控制与实验管理](01-提示的版本控制与实验管理.md) -- Prompt 具有代码与配置的双重身份。它需要版本控制，更需要实验管理：A/B 测试、效果对比、变更归因、回滚机制。将 ML 实验管理的方法论引入 prompt 工程。
- [输出质量的统计度量](02-输出质量的统计度量.md) -- "看起来不错"不是质量标准。如何定义可量化的质量维度，如何建立具有统计属性的评估基线，如何区分信号与噪声。数据科学的基本功在这里派上用场。
- [收敛性思维与提示迭代](03-收敛性思维与提示迭代.md) -- 计算数学的核心问题之一是迭代法的收敛性。Prompt 的迭代优化面临完全相同的结构性问题：多数人在没有收敛判据的情况下盲目迭代，既不知道是否在接近最优解，也不知道何时该停止。
- [数值稳定性的隐喻](04-数值稳定性的隐喻.md) -- 理论上等价的两个计算路径，在有限精度下可能产生截然不同的结果。语义上等价的两种 prompt 表述，可能导致质量上的巨大差异。如何识别和回避"数值不稳定"的 prompt 结构。
- [数据飞轮的构建](05-数据飞轮的构建.md) -- 数据飞轮不是 buzzword，是一种具体的系统架构模式。用户反馈、输出日志、人工标注如何形成正反馈闭环，冷启动问题，以及飞轮加速与停滞的判断。
- [特征工程思维在提示设计中的投射](06-特征工程思维在提示设计中的投射.md) -- Prompt 中的上下文信息选择与机器学习中的特征工程结构同构。上下文窗口是"特征预算"，信息增益视角，特征交互效应在 prompt 中的表现。
- [何时微调而非继续提示](07-何时微调而非继续提示.md) -- 一个实际的工程决策。Prompt 工程的能力天花板、微调的成本结构与风险分析、"prompt 优先、微调兜底"的决策框架。

## 本章的推演逻辑

七篇文章分为三组：

第一组（前两篇）建立工程基础设施——版本控制和质量度量是一切数据驱动优化的前提。没有版本控制，无法追溯变更；没有统计度量，无法判断变更的效果。

第二组（第三、四篇）引入计算数学的核心思维工具——收敛性分析帮助你判断迭代是否有效、何时停止；数值稳定性帮助你理解为什么"等价的"prompt 修改会产生不等价的结果。这两个概念在全书中多次作为类比出现，本章将它们从类比提升为可操作的分析框架。

第三组（后三篇）讨论系统级的数据策略——数据飞轮的构建、上下文信息的选择，以及何时从 prompt 工程升级到微调。这三篇的共同主题是：如何利用数据持续改善系统质量。

本章的数据科学思维为前面七章提供了量化工具和实验方法论。[第九章](../09-人机协作的软件过程/00-概述.md)将视角从"做什么"和"怎么做"转向"以什么方式工作"——LLM 如何改变软件开发过程本身。
