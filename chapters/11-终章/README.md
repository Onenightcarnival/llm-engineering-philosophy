# 第十一章 终章：软件工程的下一个形态

## 核心问题

软件工程从未停止演化。LLM 带来的不是又一个工具，而是又一次范式转换。这一章不讨论具体技术，而是构想未来。

## 计划文章

- **范式转换的历史节奏** -- 从结构化编程到面向对象，从瀑布到敏捷，从单体到微服务。每次范式转换都重新定义了"什么是好的软件"。LLM 是这个序列中的下一拍。
- **规格说明的回归** -- 软件开发可能正在从"编写实现"转向"编写规格说明"。当 LLM 能够将规格说明转化为实现时，开发者的核心能力从"如何实现"变为"如何精确地描述意图"。这恰恰是声明式编程的终极形态。
- **软件工程师的下一个身份** -- 不是预测谁会被替代，而是讨论能力结构的变化。系统思维、约束建模、质量判断——这些能力在 LLM 时代不是贬值了，而是升值了。
