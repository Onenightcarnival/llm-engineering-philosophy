# 第七章 数据工程视角

## 核心问题

数据科学训练带来一种独特的思维方式：对不确定性的量化、对实验方法的重视、对数据流的结构性理解。这些思维方式在 LLM 应用开发中极具价值。

## 计划文章

- **Prompt 的版本控制与实验管理** -- Prompt 是代码还是配置？无论答案是什么，它需要版本控制。更重要的是，它需要实验管理：A/B 测试、效果对比、回滚机制。将 ML 实验管理的方法论引入 prompt 工程。
- **输出质量的统计度量** -- 不能只靠"看起来不错"来判断 LLM 输出质量。如何定义度量指标，如何建立评估基线，如何做统计显著性检验。数据科学的基本功在这里派上用场。
- **数据飞轮的构建** -- 用户反馈、输出日志、人工标注——这些数据如何形成闭环，持续改善系统质量。数据飞轮不是 buzzword，它是一种具体的系统架构模式。
- **特征工程思维在 Prompt 设计中的投射** -- Prompt 中的上下文信息选择，本质上是一种特征工程。哪些信息对 LLM 的输出质量有正向影响，如何衡量，如何筛选。
- **何时该微调而非继续 prompt** -- 一个实际的工程决策。微调的成本、收益和风险分析，以及"prompt 优先、微调兜底"的决策框架。
