# 第一章 认识论：大模型的本质与边界

## 核心问题

在将大模型作为软件组件使用之前，必须先理解它到底是什么。不是从机器学习的角度（参数、损失函数、训练数据），而是从计算理论和语言哲学的角度。

这一章是全书的哲学地基。后续每一个工程决策——prompt 怎么设计、类型系统怎么用、架构怎么选、测试怎么做——都以本章确立的认识论为出发点。如果对大模型的本质认知是错的，后续再精巧的工程方法也建立在沙地上。

## 文章

- [自回归生成的本质](01-自回归生成的本质.md) -- 大模型在概率空间中执行条件采样。条件概率分解、采样策略作为设计决策、以及这个核心机制对工程实践的直接推论。
- [语言作为接口：比 API 更古老，比 API 更模糊](02-语言作为接口.md) -- 自然语言是人类最早的接口协议，也是最不精确的。形式化 API 与自然语言"契约"的结构性差异，以及模糊性的工程代价。
- [确定性与概率性的根本张力](03-确定性与概率性.md) -- 软件工程建立在确定性假设之上。当核心组件变成概率性的，哪些工程原则失效了，哪些反而变得更重要了。级联可靠性的数学约束，以及"正确"的重新定义。
- [大模型不擅长什么](04-大模型不擅长什么.md) -- 比了解能力更重要的是了解边界。精确计算、状态维护、实时性、一致性、指令遵从——这些局限是本质属性。边界认知直接指向混合架构的设计原则。

## 本章的推演逻辑

四篇文章构成一条从机制到边界的推演链：

第一篇从自回归生成的数学机制出发，确立"LLM 是概率性生成器"这个基本事实。第二篇从接口的角度分析，当系统的接口从形式化语言变为自然语言时，精确性如何被牺牲、又如何被恢复。第三篇将视角从 LLM 本身扩展到整个软件工程范式，分析确定性假设被打破后的系统性后果。第四篇划定大模型的能力边界，指向混合架构的设计原则。

本章的结论——大模型是概率性的生成器，有明确的能力边界，需要在系统层面被约束和管理——是[第二章](../02-不确定性与决策/00-概述.md)决策框架的前提。
