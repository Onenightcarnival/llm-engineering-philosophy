# 第一章 认识论：大模型的本质与边界

## 核心问题

在将大模型作为软件组件使用之前，必须先理解它到底是什么。不是从机器学习的角度（参数、损失函数、训练数据），而是从计算理论和语言哲学的角度。

## 计划文章

- **自回归生成的本质** -- 大模型不是"理解"语言，而是在概率空间中执行条件采样。这个认识如何改变我们设计系统的方式。
- **语言作为接口：比 API 更古老，比 API 更模糊** -- 自然语言是人类最早的接口协议，也是最不精确的。用它来驱动软件系统意味着什么。
- **确定性与概率性的根本张力** -- 软件工程建立在确定性假设之上。当核心组件变成概率性的，哪些工程原则失效了，哪些反而变得更重要了。
- **大模型不擅长什么** -- 比了解能力更重要的是了解边界。精确计算、状态维护、实时性、一致性——这些局限不是缺陷，是本质。
